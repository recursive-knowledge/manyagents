## M0 build decisions (conventions inherited by M1–M10)

- **Per-milestone dependency declaration.** Runtime deps are added to
  `[project.dependencies]` only when the `src/` code importing them lands, so
  `deptry src` (DEP002) and `make check` stay green at every boundary. The
  crystalline plan's M0 dep list is the *eventual* set, not an M0 requirement.
  M0 runtime dep: `python-dotenv` only.
- **Dev Python pinned to 3.12** via committed `.python-version` (un-gitignored)
  so dev == `tox envlist=py312` == CI matrix. Eliminates 3.13-drift.
- **mkdocs `exclude_docs: design/` — deliberate divergence.** Package Structure
  & Workflow says ship design docs under `docs/design/`; they DO ship in-repo,
  but are excluded from the rendered site because their mermaid + cross-refs
  fail `mkdocs build --strict`. The "why" stays in-repo, not on the docs site.
  **TODO(M10): acknowledge this explicitly** (CLAUDE.md note added; revisit if
  design docs are made strict-clean). Status: noted; finalize in M10.
  **M10 finalize (2026-05-19):** `design/` stays excluded — `mkdocs build
  --strict` is green with the M10 nav (Home + Getting Started + the 3
  `docs/guide/` stubs); the `exclude_docs: design/ archive/` block carries the
  rationale inline. Revisit only if a design doc is ever made strict-clean
  (mermaid + cross-refs); no further action this build.
- **PEP-562 lazy surface** mirrored verbatim from datasmith
  `src/datasmith/__init__.py`. `_LAZY_IMPORTS` empty at M0; each milestone adds
  its public symbols + the matching `.pyi` alias (drift-guarded by
  `tests/test_package.py::test_pyi_mirrors_runtime_surface`).
- **CI uv pin `0.6.14`** (mirrored from datasmith) verified to parse the
  `version=1, revision=3` lock generated by local uv 0.11.15 — no bump needed.
