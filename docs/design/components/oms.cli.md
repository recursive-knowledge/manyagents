## Abstract

`oms.cli` sequences `oms.core` + `oms.adapters` + `oms.capture` + `oms.forum` + `oms.distill` + `oms.bank`. It owns no domain logic — guards, schema, anti-meta, curator selection, reuse all live in modules so CLI and programmatic API cannot diverge.

## Commands

| Command | Orchestrates |
|---|---|
| `oms start [id] [--goal "..."]` | `Session.start`; `--goal` sets a soft scope (optional; agent can infer later) |
| `oms register <name>` | resolve adapter (trusted key) → `Agent` row |
| `oms <name> [args]` | `adapter.invoke` under the owned PTY; raw stream → `oms.capture` |
| `oms end [--session id]` | confirm; mark `ended`; this is one natural ★-prompt moment |

### Slash commands

| Slash | Orchestrates |
|---|---|
| `/self-distill [guidance]` | agent writes one structured **`reflection` post** (`oms.forum`, anti-meta, agent-generated); agent **proposes a ★**; human one-tap **accept/reject + confirm/override ★** (non-blocking; "unrated" valid) |
| `/discuss [@packet] [--stance agree\|disagree\|synthesize]` | **new.** agent does retrieval-before-post, then writes one stance **`reply` post** engaging a prior post |
| `/cross-distill [--server]` | invoke the **curator** (`oms.distill`; `local` default, `--server` selects hosted) over goal-scoped posts → a `distill` packet |
| `/inject [@packet]` | human-gated preview (head-100+tail-100 tokens) → inject the curated goal-scoped bundle; **write an `injections` row** (downstream-reuse tracking) |

## Design Decisions

### The human surface is one tap — **Settled (Design Principles §11)**

`/self-distill` is the only place structure is produced and it is produced by the *agent* (it already holds the session). The human sees the proposed post + a proposed ★ and taps accept/reject (and may override the ★). No JSON, no schema, no rating math touches the practitioner. This is the single constraint that keeps the open-ended loop intact while gaining swarms' discipline.

### Verbs are thin; rules live in modules — **Settled**

"Run /self-distill first!" (no posts under scope), retrieval-before-post, the inject preview, quarantine exclusion, curator selection, reuse weighting — all in `oms.forum`/`oms.distill`/`oms.bank`. The CLI renders prompts and reads taps.

### ★ prompt placement — **Settled**

Asked only at `/self-distill` accept-time and `oms end` (moments the human is already evaluating). Agent proposes; unrated is valid and common; never blocks the loop. Non-interactive mode (`OMS_NONINTERACTIVE`) → unrated + no inject (deny-by-default, Open-Q §B5).

### Curator selection — **Settled (hybrid)**

`/cross-distill` uses `OMS_CURATOR_MODE` (`local`|`server`|`auto`, `oms.utils`); `--server` overrides to hosted for one call. `auto` falls back to local if the server is unreachable, so the loop never hard-fails on curation.

### Slash interception, active-session, SIGINT — **Settled (unchanged)**

`/`-commands consumed before the wrapped agent (a non-command `/path` passes through). `~/.oms/active` holds the session; `--session` overrides. The mandatory two-stage child-agent SIGINT handler (datasmith precedent — `oms` wraps a live agent) is unchanged from the prior pass.

- **2026-06-10 — message catalog, ★ picker, and the second wave of defaults (user decisions).** (1) **`oms.utils.messages`** is now the single home of all user-facing interface text (prompts, offers, hints, one-line statuses) as named template constants — never inline at call sites (`oms.cli` + `oms._handlers` refactored onto it). Deliberately a constants module, not gettext: no translation requirement; the catalog shape is gettext-compatible if one appears. The confusing end-of-session wording was rewritten there ("this session has no saved insight yet — summarize what it learned into a reflection post?"). (2) **The commit gate on a real TTY is now `ui.pick_star`**: a 1★…5★ number line (arrows/`1-5` move, Enter commits with the selected ★, `s` commits unrated, `n`/Esc discards) with explicit "1★ = poor → 5★ = best" anchors; scripted callers (tests, `oms.testing.Simulation`) keep the typed one-liner fallback since they inject their own `input_fn`. (3) New defaults: **goal continuity** (`oms start` with no `--goal` offers the previous session's goal); **quarantine note** (one informational line, no gate); **stale-goal cross nudge** — `/cross-distill` is now offered at `oms start` when ≥ `OMS_CROSS_NUDGE_MIN` (default 3) reflections accumulated under the goal SINCE its newest bundle, which **replaces** the end-of-session cross offer (the newer-than-bundle counting is the guard against back-to-back re-runs); **injected-bundle follow-up** — when a bundle was injected into the session, the end-of-session offer becomes a guided reflection ("did it hold up?") whose guidance cites the bundle via `evidence_ref` (v1 of conflict-driven discussion; a true cross-session `disagree` reply needs goal-scoped `retrieve()`, deliberately not loosened here). - **2026-06-10 — the session-close moments moved to agent exit (user decision).** The distill offer fired only at `oms end`, but the wrapped agent exiting is the natural "I'm done working" point — users never saw the offer because they never typed `oms end`. Now `oms <agent>` fires, on a clean exit (rc 0, interactive): (1) the distill offer (same `_offer_end_distill`; a decline writes `$OMS_HOME/offers/<sid>.distill-declined` so neither agent exit nor `oms end` re-asks that session), then (2) **the end-session offer** ("agent finished — end session X now?") which on allow runs the full `oms end` path (status, ★ fallback, stash/marker cleanup, active cleared). `oms end` remains the explicit fallback for multi-agent sessions where the user keeps working.
- **2026-06-10 — `register` validates the adapter before minting (user decision).** `oms register agent` happily persisted `…/agent-001-agent` + printed a viewer URL for a name that resolves to nothing and isn't a runnable command — the "register is explicit but optional" wording never said the name had to be *real*. - **2026-06-10 — goal-first `oms start` + default goal bucket (user decision).** The spec'd form `oms start [id] [--goal "..."]` put the machine-generated token in the human slot: the id is the thing nobody should type (it has a generator, `sid.new()`), while the goal is the thing everyone has. Reversed: **`oms start [goal] [--id XXXX-XXXX]`** — the goal is the positional argument, the id is the opt-in flag (defaulting to a fresh `sid.new()` as before). Every session now carries a goal: with no goal argument, the continuity offer runs first, and if it declines (or `OMS_NONINTERACTIVE`) the session is filed under the **default bucket** — `OMS_DEFAULT_GOAL` (default `misc`, new tunable in `oms.env.example`; note text `messages.START_DEFAULT_GOAL_NOTE`). The default bucket is the catch-all, *not* a curated goal: `do_cross_distill` maps it to `scope=cross_goal` (so the "newcomer with no goal" cross-goal story stays reachable through the CLI — `scripts/simulate_story.py` story C), the continuity offer never proposes it, and the start-time goal offers (quarantine note / cross nudge / inject) skip it — its bundles are cross-goal and carry no goal, so e.g. the nudge could never converge. oms.core's "a session may have no goal" stays true at the model layer (`Session.goal` remains nullable; programmatic callers can still write `None`); the *CLI* surface now always assigns one.
