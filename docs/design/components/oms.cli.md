## Abstract

`oms.cli` sequences `oms.core` + `oms.adapters` + `oms.capture` + `oms.forum` + `oms.distill` + `oms.bank`. It owns no domain logic ‚Äî guards, schema, anti-meta, curator selection, reuse all live in modules so CLI and programmatic API cannot diverge.

## Commands

| Command | Orchestrates |
|---|---|
| `oms start [id] [--goal "..."]` | `Session.start`; `--goal` sets a soft scope (optional; agent can infer later) |
| `oms register <name>` | resolve adapter (trusted key) ‚Üí `Agent` row |
| `oms <name> [args]` | `adapter.invoke` under the owned PTY; raw stream ‚Üí `oms.capture` |
| `oms end [--session id]` | confirm; mark `ended`; this is one natural ‚òÖ-prompt moment |

### Slash commands

| Slash | Orchestrates |
|---|---|
| `/self-distill [guidance]` | agent writes one structured **`reflection` post** (`oms.forum`, anti-meta, agent-generated); agent **proposes a ‚òÖ**; human one-tap **accept/reject + confirm/override ‚òÖ** (non-blocking; "unrated" valid) |
| `/discuss [@packet] [--stance agree\|disagree\|synthesize]` | **new.** agent does retrieval-before-post, then writes one stance **`reply` post** engaging a prior post |
| `/cross-distill [--server]` | invoke the **curator** (`oms.distill`; `local` default, `--server` selects hosted) over goal-scoped posts ‚Üí a `distill` packet |
| `/inject [@packet]` | human-gated preview (head-100+tail-100 tokens) ‚Üí inject the curated goal-scoped bundle; **write an `injections` row** (downstream-reuse tracking) |

## Design Decisions

### The human surface is one tap ‚Äî **Settled (Design Principles ¬ß11)**

`/self-distill` is the only place structure is produced and it is produced by the *agent* (it already holds the session). The human sees the proposed post + a proposed ‚òÖ and taps accept/reject (and may override the ‚òÖ). No JSON, no schema, no rating math touches the practitioner. This is the single constraint that keeps the open-ended loop intact while gaining swarms' discipline.

### Verbs are thin; rules live in modules ‚Äî **Settled**

"Run /self-distill first!" (no posts under scope), retrieval-before-post, the inject preview, quarantine exclusion, curator selection, reuse weighting ‚Äî all in `oms.forum`/`oms.distill`/`oms.bank`. The CLI renders prompts and reads taps.

### ‚òÖ prompt placement ‚Äî **Settled**

Asked only at `/self-distill` accept-time and `oms end` (moments the human is already evaluating). Agent proposes; unrated is valid and common; never blocks the loop. Non-interactive mode (`OMS_NONINTERACTIVE`) ‚Üí unrated + no inject (deny-by-default, Open-Q ¬ßB5).

### Curator selection ‚Äî **Settled (hybrid)**

`/cross-distill` uses `OMS_CURATOR_MODE` (`local`|`server`|`auto`, `oms.utils`); `--server` overrides to hosted for one call. `auto` falls back to local if the server is unreachable, so the loop never hard-fails on curation.

### Slash interception, active-session, SIGINT ‚Äî **Settled (unchanged)**

`/`-commands consumed before the wrapped agent (a non-command `/path` passes through). `~/.oms/active` holds the session; `--session` overrides. The mandatory two-stage child-agent SIGINT handler (datasmith precedent ‚Äî `oms` wraps a live agent) is unchanged from the prior pass.
