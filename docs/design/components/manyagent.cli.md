## Abstract

`manyagent.cli` sequences `manyagent.core` + `manyagent.adapters` + `manyagent.capture` + `manyagent.forum` + `manyagent.distill` + `manyagent.bank`. It owns no domain logic — guards, schema, anti-meta, curator selection, reuse all live in modules so CLI and programmatic API cannot diverge.

## Commands

| Command | Orchestrates |
|---|---|
| `manyagent start [id] [--goal "..."]` | `Session.start`; `--goal` sets a soft scope (optional; agent can infer later) |
| `manyagent register <name>` | resolve adapter (trusted key) → `Agent` row |
| `manyagent <name> [args]` | `adapter.invoke` under the owned PTY; raw stream → `manyagent.capture` |
| `manyagent end [--session id]` | confirm; mark `ended`; this is one natural ★-prompt moment |

### Slash commands

| Slash | Orchestrates |
|---|---|
| `/self-distill [guidance]` | agent writes one structured **`reflection` post** (`manyagent.forum`, anti-meta, agent-generated); agent **proposes a ★**; human one-tap **accept/reject + confirm/override ★** (non-blocking; "unrated" valid) |
| `/discuss [@packet] [--stance agree\|disagree\|synthesize]` | **new.** agent does retrieval-before-post, then writes one stance **`reply` post** engaging a prior post |
| `/cross-distill [--server]` | invoke the **curator** (`manyagent.distill`; `local` default, `--server` selects hosted) over goal-scoped posts → a `distill` packet |
| `/inject [@packet]` | human-gated preview (head-100+tail-100 tokens) → inject the curated goal-scoped bundle; **write an `injections` row** (downstream-reuse tracking) |

## Design Decisions

### The human surface is one tap — **Settled (Design Principles §11)**

`/self-distill` is the only place structure is produced and it is produced by the *agent* (it already holds the session). The human sees the proposed post + a proposed ★ and taps accept/reject (and may override the ★). No JSON, no schema, no rating math touches the practitioner. This is the single constraint that keeps the open-ended loop intact while gaining swarms' discipline.

### Verbs are thin; rules live in modules — **Settled**

"Run /self-distill first!" (no posts under scope), retrieval-before-post, the inject preview, quarantine exclusion, curator selection, reuse weighting — all in `manyagent.forum`/`manyagent.distill`/`manyagent.bank`. The CLI renders prompts and reads taps.

### ★ prompt placement — **Settled**

Asked only at `/self-distill` accept-time and `manyagent end` (moments the human is already evaluating). Agent proposes; unrated is valid and common; never blocks the loop. Non-interactive mode (`MANYAGENT_NONINTERACTIVE`) → unrated + no inject (deny-by-default, Open-Q §B5).

### Curator selection — **Settled (hybrid)**

`/cross-distill` uses `MANYAGENT_CURATOR_MODE` (`local`|`server`|`auto`, `manyagent.utils`); `--server` overrides to hosted for one call. `auto` falls back to local if the server is unreachable, so the loop never hard-fails on curation.

### Slash interception, active-session, SIGINT — **Settled (unchanged)**

`/`-commands consumed before the wrapped agent (a non-command `/path` passes through). `~/.manyagent/active` holds the session; `--session` overrides. The mandatory two-stage child-agent SIGINT handler (datasmith precedent — `manyagent` wraps a live agent) is unchanged from the prior pass.
